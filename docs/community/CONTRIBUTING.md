# How to Contribute to Autarky

Thank you for your interest in contributing! This guide will help you get started.

## Our Values

1. **Openness:** Everything is shared, nothing is proprietary
2. **Safety:** No experiment is worth risking health or environment
3. **Rigor:** Document everything, test thoroughly, share data
4. **Inclusion:** All backgrounds welcome, respect all perspectives
5. **Iteration:** Start small, learn, improve, scale

## Ways to Contribute

### 1. Technical Contributions
- **Experiment replication:** Try one of our 6 experiments, document results
- **Design improvements:** Suggest better designs for components
- **Code development:** Help with simulations, control systems, Social OS
- **Documentation:** Improve guides, create tutorials, translate

### 2. Non-Technical Contributions
- **Community building:** Help organize, moderate discussions
- **Outreach:** Write articles, create videos, give talks
- **Funding:** Help find grants, manage crowdfunding
- **Connections:** Introduce us to potential collaborators

### 3. Resource Contributions
- **Equipment:** Donate or loan lab equipment
- **Materials:** Source hard-to-find materials
- **Space:** Offer workspace for experiments
- **Expertise:** Share specialized knowledge

## Getting Started

### For Technical Contributors:

1. **Start with safety:** Read [SAFETY.md](/docs/SAFETY.md)
2. **Pick an experiment:** Choose from our [6 experiments](/experiments/)
3. **Set up your workspace:** Follow the protocol
4. **Document everything:** Take notes, photos, data
5. **Share results:** Post on Discord, create GitHub issue

### For Non-Technical Contributors:

1. **Join our Discord:** [link]
2. **Introduce yourself:** Tell us about your interests/skills
3. **Find a task:** Check the "help wanted" channel
4. **Start contributing:** Even small tasks help

## Communication Channels

- **GitHub Issues:** For bugs, feature requests, questions
- **Discord:** For real-time discussion and collaboration
- **Email:** For sensitive or private matters

## Project Organization

- **Main repository:** `autarky` (this one)
- **Experiment branches:** Each experiment has its own folder
- **Design branches:** System designs are separate
- **Documentation:** All in `/docs/`

## Code/Design Standards

### For Code:
- Use clear, commented code
- Include tests where possible
- Document functions and classes
- Follow existing style in the repository

### For Designs:
- Use metric units
- Include tolerances
- Specify materials clearly
- Include safety considerations

### For Documentation:
- Use Markdown format
- Include images/diagrams where helpful
- Link to related resources
- Keep language clear and accessible

## Reporting Issues

When reporting issues:

1. Use the GitHub issue tracker
2. Include:
   - Clear description of the issue
   - Steps to reproduce (if applicable)
   - Expected vs actual behavior
   - System/environment details
   - Relevant logs or error messages

## Proposing Changes

For significant changes:

1. Discuss on Discord first
2. Create a GitHub issue outlining the change
3. If approved, create a branch for your work
4. Submit a pull request when ready
5. Address any review comments

## Recognition

All contributors will be:
- Listed in [TEAM.md](/community/TEAM.md)
- Acknowledged in papers/publications
- Invited to participate in decision-making
- Eligible for revenue sharing from any commercial applications

## Questions?

Ask on Discord or create a GitHub issue. We're here to help!
